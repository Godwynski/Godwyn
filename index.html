Learn Database Models

Explore different types of data models used in database systems

Relational Model

The relational model uses tables (also called relations) to represent data and their relationships. Each table consists of rows and columns. The primary feature of the relational model is its use of **SQL (Structured Query Language)** to manage data.

Hierarchical Model

The hierarchical model organizes data in a tree-like structure, where each record has a single parent but can have multiple children. This model is ideal for representing data that has a clear parent-child relationship, such as an organizational chart.

Network Model

The network model is similar to the hierarchical model but allows more complex relationships between records. A record can have multiple parent records, making it suitable for representing complex relationships, such as in telecommunications.

Object-Oriented Model

The object-oriented model is based on the concept of objects, which contain both data and methods. It allows the representation of complex data structures and is used in object-oriented programming environments.

Document Model

The document model stores data in document formats such as JSON, BSON, or XML. It is widely used in NoSQL databases like MongoDB and is suitable for unstructured or semi-structured data.
